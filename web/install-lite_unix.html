Installing NLTK-Lite from Source on Unix or Macintosh (OS X)

1. Installing Python

NLTK-Lite requires the Python scripting language interpreter to be installed on your machine.

Download the Python source distribution from http://www.python.org/ftp/python/2.5/Python-2.5.tgz to /tmp.

Open a terminal session, and uncompress the Python source distribution: 

	cd /tmp
	gunzip Python-2.5.tar.gz
	tar -xvf Python-2.5.tar

Compile and install the Python source distribution: 

	cd Python-2.5
	./configure
	make
	sudo make install

Check the Python installation: 

	python -V

2. Installing Numpy

NLTK also requires a Python module called Numpy, which provides a range of mathematical functions.

Download the Numpy source distribution from http://prdownloads.sourceforge.net/numpy/numpy-1.0.1.tar.gz?download to /tmp. NUMPY = $(SFNETMIRROR)/numpy wget -N -P python/unix/ $(NUMPY)/

Uncompress the Numpy source distribution: 

	cd /tmp
	gunzip numpy-1.0.1.tar.gz
	tar -xvf numpy-1.0.1.tar

Install the Numpy module: 

	cd numpy-1.0.1
	sudo python setup.py install

3. Installing NLTK-Lite

Download the NLTK-Lite source distribution from http://prdownloads.sourceforge.net/nltk/nltk_lite-0.7.3.tar.gz?download to /tmp.

Uncompress the NLTK-Lite source distribution: 

	cd /tmp
	gunzip nltk_lite-0.7.3.tar.gz
	tar -xvf nltk_lite-0.7.3.tar

Install the NLTK-Lite module: 

	cd nltk_lite-0.7.3
	sudo python setup.py install

4. Installing the NLTK-Lite Corpora Package

Download the NLTK-Lite Corpora distribution from http://prdownloads.sourceforge.net/nltk/nltk_lite-corpora-0.7.3.zip?download to /tmp.

Uncompress the NLTK-Lite Corpora distribution: 

	cd /tmp
	unzip nltk_lite-corpora-0.7.3.zip

Move the corpora to a standard directory, e.g. /usr/share/nltk_lite or /usr/local/share/nltk_lite. If you are not a system administrator, you can put this directory in your home directory (e.g. ~/nltk_lite). 

	mkdir /usr/share/nltk_lite
	mv /tmp/corpora /usr/share/nltk_lite/

Set an environment variable for the NLTK-Lite Corpora 

	export NLTK_LITE_CORPORA=/usr/share/nltk_lite/corpora/

For convenience this environment variable can be set in your $HOME/.bash_profile so it is loaded every time you start a terminal session: 

	echo "export NLTK_LITE_CORPORA=/usr/share/nltk_lite/corpora/" >> $HOME/.bash_profile

Alternatively, you can also use the function set_basedir() in the module nltk_lite.corpora: 

	[user@host ~]$ python
	Python 2.4.1 (#1, Aug  4 2005, 10:25:27)
	[GCC 3.4.3 20050227 (Red Hat 3.4.3-22.fc3)] on linux2
	Type "help", "copyright", "credits" or "license" for more information.
	>>> from nltk_lite.corpora import set_basedir
	>>> set_basedir('/some/other/path')

5. Installing WordNet (optional)

Download the WordNet source distribution from http://wordnet.princeton.edu/2.1/WordNet-2.1.tar.gz to /tmp.

Open a terminal session, and uncompress the distribution: 

	cd /tmp
	gunzip WordNet-2.1.tar.gz
	tar -xvf WordNet-2.1.tar

Compile and install the WordNet source distribution. 

	cd WordNet-2.1
	./configure
	make
	sudo make install

By default, WordNet will be installed in /usr/local. You will need to ensure that your PATH environment variable is extended to include /usr/local/WordNet-2.1/bin.

To install WordNet somewhere other than /usr/local, e.g. /Library/WordNet: 

        cd WordNet-2.1
        ./configure --prefix=/Library/WordNet
        make install

If WordNet is not installed in /usr/local, set environment variable WNHOME=/Library/WordNet.
