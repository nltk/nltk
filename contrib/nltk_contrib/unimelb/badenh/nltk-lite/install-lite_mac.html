Installing NLTK-Lite on Macintosh (OS X)

Preliminaries

Open a terminal session.

Decide on a temporary location in which to store downloaded files, eg /tmp

Set this temporary location as an environment variable 

	export TMP=/tmp

Installing Python

NLTK-Lite requires the Python scripting language interpreter to be installed on your machine.

Python is available as either a source distribution which needs to be compiled, or as a binary.

Source

Check if a C/C++ compiler, gcc, is installed: 

	gcc -v

If not, you need to download and install:

  • for Mac OS X 10.3x, Xcode 1.5 from http://developer.apple.com/ (requires registration)
  • for Mac OS X 10.4x, Xcode 2.1 from http://developer.apple.com/ (requires registration)

Mount the dmg, and follow the OSX Packager installation instructions.

If gcc is installed, then download the Python source distribution from http://www.python.org/ftp/python/2.4.1/Python-2.4.1.tgz to $TMP.

Uncompress the Python source distribution: 

	cd $TMP
	gunzip Python-2.4.1.tar.gz
	tar -xvf Python-2.4.1.tar

Compile and install the Python source distribution: 

	cd Python-2.4.1
	./configure
	make
	sudo make install

Check the Python installation: 

	python -V

Binary

Download the appropriate Python binary from http://undefined.org/python/MacPython-OSX-2.4.1-1.dmg to $TMP.

Mount the dmg, double click and follow the OSX Packager install instructions.

Then modify the system's linkages from the standard Python 2.3.x to the new version: 

	sudo ln -s /usr/local/bin/python /usr/bin/python

Check the Python installation: 

	python -V

Installing Numarray

CURRENTLY NUMARRAY IS NOT DISTRIBUTED AS A BINARY FOR MAC OSX

NLTK also requires a Python module called Numarray, which provides a range of mathematical functions.

Download the Numarray source distribution from http://prdownloads.sourceforge.net/numpy/numarray-1.3.3.tar.gz?download to $TMP.

Uncompress the Numarray source distribution: 

	cd $TMP
	gunzip numarray-1.3.3.tar.gz
	tar -xvf numarray-1.3.3.tar.gz

Install the Numarray module: 

	cd numarray-1.3.3
	sudo python setup.py install

Installing NLTK-Lite

Download the NLTK-Lite source distribution from http://prdownloads.sourceforge.net/nltk/nltk_lite-0.2.tar.gz?download to $TMP.

Uncompress the NLTK-Lite source distribution: 

	cd $TMP
	gunzip nltk_lite-0.2.tar.gz
	tar -xvf nltk_lite-0.2.tar

Install the NLTK-Lite mododule: 

	cd nltk_lite-0.2
	sudo python setup.py install

Installing the NLTK-Lite Data Package

Download the NLTK-Lite Data distribution from http://prdownloads.sourceforge.net/nltk/nltk_lite-corpora-0.2.zip?download to $TMP.

Uncompress the NLTK-Lite Data distribution: 

	cd $TMP
	unzip nltk_lite-corpora-0.2.zip

Create a directory to hold the NLTK-Lite corpus set. A common path for this directory is /usr/share/nltk. If you are not a system administrator, you can put this directory in your home directory (e.g. ~/nltk). 

	mkdir /usr/share/nltk

Uncompress each corpus that you'd like to use to into the NLTK corpus directory that you created, eg: 

	unzip -d /usr/share/nltk/ brown.zip

For convenience, a recursive uncompression can be performed by: 

	cd $TMP/nltk_lite-corpora-0.2
	ls | grep zip | while read i; do unzip -d /usr/share/nltk $i; done

Set an environment variable for the NLTK-Lite Data 

	export NLTK_LITE_CORPORA=/PATH/TO/DATA

For convenience this environment variable can be set in your $HOME/.bash_profile so it is loaded every time you start a terminal session: 

	echo "export NLTK_LITE_CORPORA=/PATH/TO/DATA" >> $HOME/.bash_profile

Alternatively, you can also use the function set_basedir() in the module nltk_lite.corpora: 

	[user@host ~]$ python
	Python 2.4.1 (#1, Aug  4 2005, 10:25:27)
	[GCC 3.4.3 20050227 (Red Hat 3.4.3-22.fc3)] on linux2
	Type "help", "copyright", "credits" or "license" for more information.
	>>> from nltk_lite.corpora import set_basedir
	>>> set_basedir('/PATH/TO/DATA/')
